High Quality Real Time Rendering

Abstract

In this Project 3, we are implementing a high quality real time rendering enging using OpenGl and GLSL shader language. The first part we implement is the scene loader, mesh loader and the FPS like control system, which require us to write a generalized .obj file loader that can handle multiple object loading in the same file. For our shadow and lighting, we implemented both direct light which will generate sampler2DShadow map and point light which will generate samplerCubeShadow map. We also implemented SSAO (screen-based ambient occlusion) to assist the shadowing to make the scene look more realistic. For the environment map, we have implemented both normal environment map and irradiance environment map, which can not only give the scene a environment but can also contribute to the reflection and diffuse lighting of the scene.

Compile & Run

Our program is setup in Xcode on MacOS. So if you want to compile the code you might need Xcode to do that. In our provided code there's already a CSE163-final.xcodeproj file. Open Xcode and select this file so that you can open this project as a whole. After that, when all the project has been setup in Xcode, the code should be automatically compiled and you can than run it.

Or if you are using the appropriate system, you can download the executable (including the resources like models, scene files, environment maps and so on) from this website directly. Simply hit the button in the left bottom to download.

When starting the program, we requires the user to specify the scene file. To do that you may want to have a scene file setup. You can either download our resource archive (including the .obj files), or follow the write-up downward to write your own scene files.

When you have your scene file, use this command: 

$ ./CSE163-final basic.scene
to start the program. This will start the program with the basic.scene. Note that all the .obj files that are specified in basic.scene must also be placed within the same directory.

In the source code we provided on TED, there's a directory called resource. If you want to test run any of the scene files, environment maps, simply copy the files to the directory.
